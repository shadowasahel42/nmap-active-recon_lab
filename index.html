lab notebook — network enumeration

Enumeration is the art of asking a system enough small questions until it tells you the truth.

This project documents a real enumeration exercise carried out with Nmap against a small, intentionally vulnerable lab network. It is written for people who have never run a port scanner before, and it walks through every command exactly as it was executed — mistakes, typos, and dead ends included.

What this lab covers

The exercise moves through the same funnel an enumeration phase usually follows in practice: figure out where you are on the network, find out who else is there, work out what those hosts are running, and then push far enough into one of them to see what a misconfiguration actually exposes. Nothing here is theoretical — every terminal block in the walkthrough is the unedited output of a command that was actually run.

-sn

Host discovery

Sweeping a /24 to work out which of 256 possible addresses are actually occupied by a live machine.

-sV -A

Service & version detection

Turning a bare list of open ports into a picture of what software is listening behind each one.

-O

OS fingerprinting

Using subtle differences in how a TCP/IP stack responds to guess the operating system underneath.

--script smb-*

SMB enumeration

Using targeted NSE scripts to pull usernames and share permissions from a Samba service.

Prerequisites

You don't need prior Nmap experience to follow along, but the walkthrough assumes:

  • A basic comfort with a Linux terminal — you don't need to know every command, but you shouldn't be seeing a shell prompt for the first time.
  • An isolated lab network you're authorized to scan. Every command here was run against machines the author owns and controls, and the same should be true for anyone following along.
  • Kali Linux, or any distribution with Nmap 7.x installed, if you intend to reproduce the commands rather than just read them.
Before you scan anything

Only run these commands against systems you own or have explicit written permission to test. Port scanning networks you don't control can be illegal even when no damage is done, because the act of probing itself is what most computer-misuse laws regulate.
